What it is

A single-user menu-bar / system-tray app that polls Atlassian JSM (and optionally Microsoft Graph) on behalf of the signed-in user and shows ticket / SLA state. It stores one secret - the user's Atlassian API token - plus, if Microsoft 365 is connected, Graph tokens. Everything else is non-secret configuration. All network calls run in the main process; the UI windows cannot make outbound requests.

Release integrity

Builds are produced by GitHub Actions and gated so a release cannot ship without human sign-off:

  • Every release requires manual approval via a protected GitHub environment (required reviewer) before any bytes are published.
  • Build actions are pinned to commit hashes, so a tampered action cannot enter the pipeline.
  • Branch and release-tag rulesets block force-push and deletion, so history and published releases cannot be rewritten.
  • Auto-updates are integrity-checked (SHA-512) against the signed-by-transport release manifest before install.

Known, accepted risks (stated plainly)

The app is currently distributed unsigned on both platforms, so first launch shows the OS "unidentified developer" / SmartScreen prompt. Auto-update integrity therefore rests on the release controls above rather than an embedded code signature. The complete fix is code-signing certificates (Apple Developer ID + a Windows certificate); the compensating controls are in place until then. Two further inherent risks are documented: any malware already running as the same user can ask the OS keystore to decrypt stored secrets (true of every app that uses the OS keystore; mitigated by EDR + disk encryption), and the nowtify:// sign-in callback scheme could be claimed by another app (mitigated by PKCE + state; worst case is a sign-in denial of service).
